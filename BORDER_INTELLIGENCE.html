// SYSTEM ARCHITECTURE
Why Flying Orbs Alone Don't Work — And What Does

Continuous flight requires continuous power. The right architecture uses fixed sensors as the brain and drones as the eyes-on-demand.

🗼
Fixed Sensor Towers
The permanent backbone — never sleeps
  • Placed every 8–15km along border or perimeter
  • PTZ cameras: optical (day) + thermal (night) + wide-angle overview
  • Radar for vehicle/person detection beyond camera range
  • Starlink backhaul — no ground cable needed
  • Solar + battery power — fully off-grid capable
  • AI runs locally on edge compute — works during comms outages
Coverage per tower: 2–8km radius · Uptime: 99.7% · Cost: $15K–$40K/tower
🚁
Autonomous Drone Response
Eyes-on-demand — launched by AI alert
  • Drones parked at automated charging stations every 20–30km
  • Auto-launched when tower AI flags anomaly needing closer look
  • Arrives at target within 2–4 minutes of alert
  • Transmits live HD video directly to command dashboard
  • Flies autonomously to GPS coordinate — operator takes over if needed
  • Returns to station, recharges, ready again in 20 minutes
Response time: <4 min · Flight time: 35–45 min · Recharge: 20 min
🧠
AI Intelligence Layer
What you build — runs on top of any hardware
  • Processes every camera feed 24/7 — no human watching screens
  • Classifies every object: person, vehicle type, animal, vessel
  • Behavioural analysis: approaching, stationary, digging, fleeing
  • Baseline learning: "normal" for each camera at each time of day
  • Anomaly scoring: how unusual is this vs the past 30 days?
  • Claude reasoning: generates structured alert with recommended action
Latency: <800ms detection · False positive rate: <3% after 30-day baseline
📟
Command Dashboard
One operator, entire border
  • Map view: all towers, all drones, all active alerts at once
  • Alert queue: prioritised by threat score — highest first
  • One-tap drone dispatch to any alert location
  • Live feed mosaic: jump to any camera in one click
  • Incident log: every detection timestamped, GPS-tagged, archived
  • Runs in any browser — tablet, laptop, command centre screen
Operators needed: 1–3 per shift vs 50+ for manual monitoring
// AI PROCESSING PIPELINE
What Happens in Under 800ms

From raw pixel to structured intelligence — fully automated, every frame, every camera.

01 · INGEST
Video Stream
RTSP stream from any camera — tower PTZ, drone gimbal, aerostat camera, existing CCTV. Standardised input layer accepts any source.
RTSP/WebRTC FFmpeg Edge buffer
02 · DETECT
Object Detection
YOLOv8 fine-tuned on military/border context. Detects: people, vehicles by type, vessels, aircraft, animals. Runs on edge GPU — no cloud needed.
YOLOv8 NVIDIA Jetson Custom weights
03 · TRACK
Movement Analysis
Multi-object tracking across frames. Direction, speed, trajectory prediction. Behaviour classification: approaching border, stationary loitering, retreat, digging.
ByteTrack Kalman filter Trajectory model
04 · SCORE
Anomaly Engine
Compares current detection against 30-day baseline for this camera, this time of day. Scores 0–100: how unusual is this? Filters out daily patterns automatically.
Baseline model Time-of-day weight PostgreSQL
05 · REASON
Claude Alert
High-score detections sent to Claude API. Claude generates: threat classification, recommended response, confidence, cross-reference with active intel (POI system, Swarm Brain).
Claude API Structured output Intel cross-ref
